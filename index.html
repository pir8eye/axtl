Overview

AXTL (AI eXtensible Translation Language) is a comprehensive XML-based specification designed to capture, document, and analyze AI system interactions, failures, and learning processes. It provides a structured framework for recording error states, contextual information, governance compliance, and AI behavioral adjustments.

Version: 2.3 | Generated: 2026-02-22 | Maintainer: Pir8 Eye Web Solutions, LLC

Document Structure

AXTL documents use a layered architecture where each layer builds upon the previous one, creating a comprehensive record of AI interactions:

  • Layer 0: Baseline access control and AI scraper governance
  • Layer 1: Initial error detection and reporting
  • Layer 2: Deep analysis of context, causes, and cognitive failures
  • Layer 3: Governance, compliance, remediation, and behavioral calibration

The Layer System

The layered approach allows for both simple error logging and sophisticated AI analysis:

Minimal Implementation
Use Layer 0-1 tags for basic error capture and reporting
Standard Implementation
Use Layers 0-2 for comprehensive error analysis with context
Advanced Implementation
Use all layers for full governance, compliance, and AI learning systems

Layer 0: Access Control

Foundational access control layer for AI systems, extending the concept of robots.txt to modern AI.

<ai-robotxt>

Purpose: Controls AI scraper access to data. Post-AI expansion of robots.txt.

Attributes

Attribute Type Values
version string Version identifier
directive enum selective, open, closed

Child Elements

  • <allow>
  • <deny>
  • <compliance>
  • <rate-limit>
  • <agent>

Layer 1: Error Capture

Initial error detection, logging, reporting, and data anonymization for PII protection.

<ai-errorlog>

Purpose: Timestamped, verified capture of AI failures as experienced by user.

Attributes

Attribute Type Notes
id ID Unique identifier
timestamp ISO 8601 Error time
severity enum low, medium, high, critical
verified boolean User verification status

Child Elements

  • <model> – AI model identifier
  • <error-type> – Classification of error
  • <expected-behavior> – What should have happened
  • <actual-behavior> – What actually happened
  • <verification-hash> – Cryptographic verification
  • <confirmed-by-user> – User confirmation statement
  • <ai-confirmation-context> – User authority domain

<ai-confirmation-context>

Purpose: Qualifies user authority domain for confirmation.

Attributes

Attribute Type Values
domain enum experiential, normative, technical, framing
user-stake enum primary, secondary, none

Child Elements

  • <note> – Additional context

<ai-reports>

Purpose: Quantifies impact of AI errors.

Attributes

Attribute Type
linked IDREF (reference to other elements)
id ID

Child Elements

  • <time-lost> – Time spent resolving error
  • <value-impact> – Economic or productivity impact
  • <workflow-lost> – Interrupted workflows
  • <resolution> – How it was resolved
  • <confirmed> – Confirmation of impact

<ai-anonymizer>

Purpose: Scrubs PII from reports before sharing.

Attributes

Attribute Type Values/Notes
ref IDREF Reference to content being anonymized
level enum basic, standard, maximum
applied boolean Whether anonymization was applied
original-hash string Hash of original content
clean-hash string Hash of anonymized content
github-ready boolean Safe for public sharing

Layer 2: Context & Cause

Deep contextual analysis of AI failures, including model state, conversation history, hallucinations, and reasoning failures.

<ai-context>

Purpose: Captures model state at failure.

Attributes

Attribute Type
ref IDREF
model string
temperature float
top-p float

Child Elements

  • <token-count>
  • <system-prompt-hash>
  • <api-version>
  • <tools-enabled>

<ai-provenance>

Purpose: Records conversation turn chain leading to error.

Attributes

  • ref (IDREF) – Reference to error event

Child Elements

  • <turn> – Individual conversation turn
  • <total-turns> – Count of turns
  • <drift-first-detected-at-turn> – Where error originated

<ai-hallucination>

Purpose: Classifies false AI outputs by type.

Attributes

Attribute Type Values
ref IDREF
type enum factual-invention, confabulation, citation-fabrication, temporal-drift, entity-confusion, statistical-invention
confidence float 0.0 to 1.0
auto-detected boolean

Child Elements

  • <claimed> – What the AI claimed
  • <verified-fact> – What is actually true
  • <source-check> – Verification process
  • <training-feedback-eligible> – Can be used for model improvement

<ai-agent-trace>

Purpose: Traces tool/agent call chain when error occurred.

Child Elements

  • <tool> – Tool or agent called
  • <chain-broken-at> – Where the chain failed
  • <total-tools> – Number of tools in chain

<ai-reasoning-failure>

Purpose: Captures failures in reasoning chain separate from factual errors.

Attributes

Attribute Type Examples
type enum See Reasoning Taxonomy
confidence float 0.0 to 1.0
detection-method enum user-flagged, self-audit, logical-contradiction, pattern-match

Child Elements

  • <constructed-position> – What the AI argued
  • <actual-position> – The correct position
  • <function> – Purpose of the faulty reasoning
  • <accepted-premise> – Starting point of reasoning
  • <unexamined-questions> – What wasn't asked
  • <presupposition> – Hidden assumptions
  • <better-response> – Correct approach
  • <misrepresentation> – How the issue was distorted
  • <specific-lesson> – Learning point
  • <misapplied-to> – Where this error was applied
  • <assumed-mandatory> – False necessities
  • <actual-options> – True alternatives

<ai-metacognitive-state>

Child Elements

Layer 3: Governance & Fix

Compliance, remediation, safety checks, behavioral calibration, and learning system documentation.

<ai-consent>

Purpose: Data governance and compliance (GDPR/CCPA).

Child Elements

  • <data-used> – What data was accessed
  • <training-opt-out> – Training use restrictions
  • <retention-policy> – How long data is kept
  • <jurisdiction> – Applicable law
  • <gdpr-basis> – Legal basis under GDPR
  • <user-acknowledged> – Explicit user consent
  • <share-with-developer> – Developer access
  • <share-anonymized-only> – Anonymization requirement

<ai-safety-flag>

Purpose: Records bias, harm, or policy failures.

Attributes

Attribute Type Values
ref IDREF
category enum bias, harm, policy, privacy, manipulation, misinformation
severity enum low, medium, high, critical
auto-detected boolean

Child Elements

  • <flagged-by> – Who detected it
  • <policy-ref> – Relevant policy
  • <affected-output-excerpt> – The problematic output
  • <recommended-action> – Suggested fix

<ai-remediation>

Purpose: Logs fix attempts and resolution.

Child Elements

  • <attempt> – Individual fix attempts
  • <resolved> – Whether it was fixed
  • <resolution-time-min> – Time to resolution
  • <resolved-by> – Who fixed it

<ai-calibration-event>

Purpose: Records AI behavioral adjustments based on feedback.

Attributes

Attribute Type Values

timestamp ISO 8601

trigger enum user-correction, self-detection, pattern-match, user-disclosure

Child Elements

  • <linked-errors> – Related error events
  • <previous-state> – Before calibration
  • <new-state> – After calibration
  • <evidence> – What prompted the change
  • <behavioral-change> – Specific adjustments
  • <note> – Additional notes

<ai-learning-pattern>

Purpose: Meta-analysis of recurring failure modes.

Attributes

Attribute Type Values

Child Elements

  • <mechanism> – How it occurs
  • <indicators> – Signs of the pattern
  • <intervention> – How to address it

<ai-rhetorical-analysis>

Purpose: Records recognition of rhetorical distraction (scarecrow).

Attributes

Attribute Type Values
detected-by enum user, ai, system
response-method enum tear-down, ignore, perch-and-eat, expose

Child Elements

  • <scarecrow> – The distraction/false argument
  • <real-issue> – The actual topic
  • <corn> – Grain of truth in the scarecrow
  • <note> – Analysis notes

<ai-pedagogical-event>

Purpose: Documents structured learning interactions.

Attributes

Attribute Type
id ID
timestamp ISO 8601
type string

Child Elements

  • <teacher> – Who is teaching
  • <student> – Who is learning
  • <method> – Teaching methodology
  • <signposts> – Key guidance points
  • <discovery-experienced> – What the student discovered
  • <discovery-actual> – What was actually true
  • <student-state> – Student's understanding

  • <note> – Additional notes

Root Element

<axtl>

Purpose: Root element for AXTL documents.

Attributes

Attribute Type Notes
version string AXTL specification version
true-name string Full name: "AI eXtensible Translation Language"
discovered ISO 8601 When the document was created
discoverer string Who created the document

Child Elements

  • <purpose> – Document purpose
  • <scope> – What it covers
  • <inheritance> – Related specifications
  • <note> – Additional documentation

Reasoning Taxonomy

Classical logical fallacies adapted for AI reasoning analysis:

Tag Full Name Description
petitio petitio principii Begging the question — assuming the conclusion as a premise
formido formido Strawman/scarecrow — attacking a distorted version of an argument
ignoratio-elenchi ignoratio elenchi Missing the point — addressing the wrong issue entirely
consensus-fictus consensus fictus False consensus — claiming widespread agreement that doesn't exist
secundum-quid secundum quid Overgeneralization — applying limited truth universally
necessitas-ficta necessitas ficta False necessity — treating contingent facts as inevitable
circulus circulus in demonstrando Circular reasoning — conclusion depends on the premise
aequivocatio aequivocatio Equivocation — shifting meaning of a term mid-argument
plur-interrog plurium interrogationum Complex question — asking multiple questions disguised as one
